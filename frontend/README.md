## рҹҢҚ The Problem

The climate crisis is **the defining challenge of our generation** вҖ” and the window to act is narrowing.

| Statistic | Source |
|-----------|--------|
| рҹҢЎпёҸ Global average temperature is **1.2В°C above pre-industrial levels** | IPCC AR6, 2021 |
| рҹ’Ё The average human emits **~4.7 tonnes of COвӮӮ per year** | World Bank, 2022 |
| рҹҮ®рҹҮі India's per-capita emissions: **1.9 t/year** вҖ” but rapidly rising | IEA, 2023 |
| рҹҮәрҹҮё US per-capita emissions: **14.5 t/year** вҖ” nearly 3Г— the global avg | EPA, 2023 |
| рҹ”Ҙ We have **< 7 years** to halve global emissions to stay under 1.5В°C | IPCC, 2023 |
| рҹ‘Ө **Individual consumption** accounts for up to **72% of global emissions** | EDGAR, 2022 |

**The painful reality:** Most people have absolutely no idea how much carbon they emit вҖ” or *where* it comes from. Existing "carbon calculators" are one-time questionnaires that produce a number and then disappear. There is no accountability loop, no behavioural science, no intelligence, and no joy.

> *"You can't manage what you don't measure."* вҖ” Peter Drucker

## рҹҢұ The Solution

**Verdant** is a full-stack, AI-powered **Carbon Intelligence Platform** that transforms abstract climate guilt into concrete, daily, gamified action.

Unlike static carbon calculators, Verdant is a **living system** вҖ” it learns from your activities, predicts future emissions, celebrates progress with XP and badges, and connects you to a community of warriors fighting for the same goal.

## вңЁ What Makes Verdant Unique

### 1. рҹ§  Gemini-Powered Intelligence вҖ” Not Just a Calculator
Every user gets a **personal AI carbon advisor** (powered by Google Gemini 1.5 Flash) that analyses their activity patterns and generates 4 insight types: **Tips**, **Warnings**, **Achievements**, and **Month-end Predictions** вҖ” each backed by IPCC/EPA emission data. A built-in rate-limiter (12 req/min per IP) and prompt sanitizer protect the endpoint.

### 2. рҹҺ® Eco Warrior Gamification вҖ” Arena-Level Engagement
The Challenges page is designed like a competitive gaming arena with **live countdown timers**, **animated XP bars**, **hexagonal SVG badge rarity system** (Common вҶ’ Legendary), a **10-warrior global leaderboard** with animated staggered entrance, and spring-physics card animations.

### 3. рҹҢҗ Three.js Immersive 3D вҖ” Science Meets Art
Every major page uses a dedicated Three.js scene. The centrepiece is a **physically-rendered Earth Globe** on the home page with a **2048Г—1024 fBm fractal terrain** (multi-octave noise, elevation-graded biomes, polar ice caps), **per-pixel roughness map** (shiny oceans, matte land), **procedural cloud layer**, **3 000-star field**, and **orbital rings** вҖ” making the globe look like a real satellite render. Secondary pages use CSS-based aurora bokeh gradients for zero GPU overhead.

### 4. рҹ“җ Science-Grade Emission Factors
All calculations use real peer-reviewed emission factors from **IPCC AR6**, **EPA eGRID**, and **IEA** вҖ” covering 24 sub-categories across Transport, Food, Energy, and Lifestyle. Not guesses. Not averages. Calibrated science.

### 5. рҹҸ° Personal Carbon Sanctuary (Profile)
The Profile page is a full **personal carbon command centre**: a vertical milestone timeline with Framer Motion animations, **inline-editable goals** with animated progress bars, toggle-rich settings, a **30-day colour-coded history chart**, and a destructive "Delete All Data" guard with type-to-confirm UX.

### 6. рҹ“Ҡ Recharts Intelligence Dashboard
The Insights page combines real AI analysis with **4 interactive Recharts visualisations**: weekly trend area charts, category donut breakdowns, daily bar comparisons, and a comparative world benchmark вҖ” all driven by the same data layer.

### 7. рҹҺЁ Dual Design Identity
Verdant has **two distinct design personalities**: a cool dark-sci-fi palette (`#00E676` neon green) for the main platform, and a **warm amber sanctuary** palette (`#FF8F00`) for the Profile page вҖ” reflecting the emotional shift from performance to reflection.

### 8. вҷҝ Accessibility-First
Every interactive element has unique `id` attributes for browser testing, ARIA `role` and `aria-checked` on toggles, keyboard navigation support, and high-contrast colour ratios throughout. All Three.js canvases are wrapped with `role="img"` and descriptive `aria-label` attributes.

### 9. рҹ”’ Security-Hardened API
The `/api/insights` route includes in-memory **IP-based rate limiting** (12 req/min), **prompt sanitization** (strips HTML/SQL injection attempts), **input length caps**, and strict **Content-Security-Policy** headers set globally via `next.config.js`.

## рҹ”¬ Carbon Calculation

Verdant's emission engine is grounded in peer-reviewed science, not estimation.

### Emission Factor Sources
| Source | Used For |
|--------|----------|
| **IPCC AR6 (2021)** | Transport emission factors (car, flight, train, bus) |
| **EPA eGRID (2023)** | Electricity grid emission factors (0.233 kg COвӮӮ/kWh) |
| **IEA World Energy Outlook** | Natural gas, LPG, heating oil factors |
| **Our World in Data / Poore & Nemecek (2018)** | Food system lifecycle emissions |
| **Carbon Trust (2023)** | Lifestyle and consumer goods factors |

### Core Formula
```
emissions_kg = activity_value Г— emission_factor_kgCO2_per_unit
```

### Emission Factors Used

## рҹӨ– AI Integration (Google Gemini)

### How It Works

```
User logs activities
       вҶ“
Zustand store accumulates up to 20 recent activities
       вҶ“
User visits /insights OR sends a chat message
       вҶ“
POST /api/insights
  body: { activities[], userQuestion, summary }
       вҶ“
Gemini 1.5 Flash receives:
  - System context: "You are an expert carbon analyst..."
  - User activities: JSON array of recent logs
  - User's carbon summary: totals by category
  - Optional user question: free-form text
       вҶ“
Gemini returns structured JSON:
  { insights[], chatResponse, summary }
       вҶ“
Frontend renders insight cards + chat response
```

### Prompt Engineering

The system prompt instructs Gemini to:
1. **Quantify everything** вҖ” "this saves 2.3 kg COвӮӮ/week"
2. **Cite sources** вҖ” reference IPCC/EPA methodology
3. **Prioritise** вҖ” high-impact, low-effort changes first
4. **Be specific** вҖ” no generic advice, personal to the user's data
5. **Return valid JSON** вҖ” structured for type-safe parsing

### Insight Types

| Type | Trigger Condition | Example |
|------|-------------------|---------|
| `tip` | Opportunity identified in user's patterns | "Switch to cold wash: save 0.3 kg/load" |
| `warning` | Category 30%+ above baseline | "Transport up 32% this week" |
| `achievement` | Notable improvement detected | "Food carbon down 15% вҖ” great!" |
| `prediction` | Projection based on current trajectory | "On track for 382 kg this month" |

### Fallback System
When the Gemini API key is not configured (or rate-limited), Verdant serves **4 rich pre-crafted fallback insights** and keyword-aware chat responses, ensuring the app is fully demonstrable without any API setup.

## рҹ“Ӣ Assumptions Made

1. **Emission factors are global averages.** The electricity emission factor (0.233 kg COвӮӮ/kWh) represents a weighted global grid mix. In reality, this varies significantly by country вҖ” India's grid is ~0.71 kg/kWh, France's is ~0.04 kg/kWh. Future versions will localise this via IP detection.

2. **Food emissions represent "cradle-to-retail" lifecycle.** We use the Poore & Nemecek (2018) median values, which include land-use change, on-farm emissions, and processing вҖ” but not the "last mile" of retail transportation or cooking energy.

3. **All users have identical emission budgets by default.** The 400 kg/month budget is the IPCC Paris-compatible per-capita target for a 1.5В°C pathway (approximately 5 tonnes/year). A future onboarding flow could personalise this based on location and household size.

4. **The leaderboard is local/mock in the current version.** All leaderboard data is generated client-side. A real multi-user leaderboard would require a database layer (e.g., Supabase/PostgreSQL).

5. **XP and gamification data persist per-device.** The Zustand store uses `localStorage` for persistence. Switching devices resets progress. A cloud sync layer would be needed for production.

6. **Flight emissions use a fixed radiative forcing multiplier of Г—1.0 (not Г—2).** Some methodologies apply a Г—2 multiplier to account for high-altitude warming effects. We chose the conservative IPCC base estimate to avoid alarming users with factors that are still scientifically debated.

7. **AI insights are generated fresh per session.** There is no caching of Gemini responses вҖ” each `/insights` page load triggers a fresh API call. For cost optimisation, a Redis/KV cache layer (TTL: 4 hours) is planned.
